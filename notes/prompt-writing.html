Prompt写作的几个误区

2026-07-10 实战技巧

很多人刚开始写 Prompt 时，总觉得"把话说得越高级越好"，于是写出一长串抽象描述，比如"请帮我写一段很有深度、打动人心的文案"。模型面对这样的指令往往也摸不着头脑——"深度"和"打动人心"对它来说太模糊了。更常见的情况是，指令不够具体：你没有说明目标读者是谁、语气是活泼还是正式、需要多长篇幅，结果得到的答案离预期差很远。写 Prompt 的第一步，其实是把需求拆成模型能听懂的小块。

第二个常见误区是不给例子。我们教别人做事时都知道"示范一下"最有效，但面对模型却常常忘了这招。比如你直接说"请用简洁风格改写下面这段话"，模型对"简洁"的理解可能和你不一样。如果你先给它一两个改写的例子，它就能迅速抓住你想要的节奏和用词。例子是最低成本的校准工具，尤其是想要固定格式、固定语气或固定输出结构时，它往往比长篇大论的规则更管用。

第三个坑是一次塞太多任务。有人写 Prompt 像在列待办清单："先总结这篇文章，再翻译一遍，然后写一段评论，最后生成三个标题"。模型确实能处理多步骤，但任务越多，越容易遗漏、跑偏或互相干扰。更稳妥的做法是把复杂流程拆成几次对话，或者先让它完成第一步，确认结果后再进入下一步。好的 Prompt 不是一次性把大脑里所有想法倒出来，而是像搭积木一样，一步一步引导模型走到你想要的地方。
